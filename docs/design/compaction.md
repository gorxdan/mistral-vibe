# Design Spec — compaction-pipeline

**Effort:** L → revised **L**  |  **Verdict:** `sound_with_fixes`  |  **Feasible:** True  |  **Depends on:** none

## Current state
Vibe has exactly two context-shaping mechanisms today, and they sit at opposite extremes:

1. Per-message tool-result truncation at ingest. `AgentLoop._apply_tool_result_budget` (vibe/core/agent_loop.py:1738-1756) caps a SINGLE tool result to `MAX_TOOL_RESULT_CHARS = 100_000` (agent_loop.py:166) keeping head 3/4 + tail 1/4, called once in `_handle_tool_response` (agent_loop.py:1767) when the result is first appended. It never revisits already-stored messages and has no notion of total context.

2. Full-summary auto-compaction. `AutoCompactMiddleware.before_turn` (vibe/core/middleware.py:100-112) compares `stats.context_tokens >= active_model.auto_compact_threshold` and returns `MiddlewareAction.COMPACT`. `_handle_middleware_result` (agent_loop.py:928-936) routes that to `_run_compaction` -> `compact()` (agent_loop.py:2141-2208), which makes a whole extra LLM call to summarize, then `self.messages.reset([system_message, compaction_context_message])` (agent_loop.py:2191) collapsing the entire history to system + one synthetic user message, then `stats.context_tokens = 0` (agent_loop.py:2197) and `_reset_session()`. This is the nuclear option: a paid LLM round-trip, total cache bust, loss of all verbatim recent turns.

There is NOTHING between these two. No stage trims a band of old history when context is moderately full; no stage compresses the oldest tool results in place; nothing runs cheapest-first.

The middleware machinery that the new stages must live in:
- `MiddlewarePipeline.run_before_turn` (middleware.py:252-266): iterates middlewares in registration order. INJECT_MESSAGE results accumulate into a combined injected user message; STOP and COMPACT SHORT-CIRCUIT (return immediately at middleware.py:258-259). CONTINUE falls through. Crucially there is no action meaning "I edited the history in place, keep going" — a shaper must mutate `context.messages` directly and return `MiddlewareResult()` (CONTINUE).
- `ConversationContext` (middleware.py:28-33) carries `messages` (the LIVE `self.messages`, built by `_get_context` at agent_loop.py:984-987), `stats`, `config`. So `context.messages` IS the canonical `MessageList`; mutating it mutates real history.
- `MessageList` (vibe/core/types.py:520-597): `reset(new)` replaces contents silently and fires reset hooks; `silent()` context manager suppresses observer notifications; supports `__getitem__`/slice/`__len__`. No in-place item-replace primitive exists besides index assignment on `_data` (used privately by `update_system_prompt`, types.py:567).
- `_setup_middleware` (agent_loop.py:871-909) clears and registers: TurnLimit, PriceLimit, TokenLimit (conditional), then `AutoCompactMiddleware()` at line 884, ContextWarning (conditional), then the ReadOnly plan/chat middlewares. This is the single registration site (also called at 353, 869, 2292).
- `_messages_for_backend` (agent_loop.py:1804-1812) returns `self.messages` unchanged unless the model lacks image support (then strips images into copies). It does NOT re-shape — shaping has already happened in middleware against `self.messages`.

Token accounting:
- `stats.context_tokens` is set ONLY after a real LLM turn: `_update_stats` (agent_loop.py:1985-1993) sets `context_tokens = usage.prompt_tokens + usage.completion_tokens`. After compaction it is forced to 0 (agent_loop.py:2197) with the comment that it's unknown until the next call. So at the FIRST `before_turn` of a fresh prompt, and immediately after a compaction, `context_tokens` is stale (last turn's value) or 0 — a shaper that triggers on it alone is one turn behind.
- A local estimator already exists: `approx_token_count(text)` (vibe/core/utils/tokens.py:9-10, ~len/4) and `truncate_middle_to_tokens(text, max_tokens)` (tokens.py:13-29, head+tail with marker). `vibe/core/compaction.py` already imports both and defines `COMPACT_USER_MESSAGE_MAX_TOKENS = 20_000`.

Cache behavior: the generic backend does NOT manage cache breakpoints — it relies on the provider auto-caching a stable prompt prefix. It only MEASURES hits: `cached = details.get("cached_tokens") ...` (vibe/core/llm/backend/generic.py:199) feeding `LLMUsage.cached_tokens` (types.py:357). Editing any message busts every cached token from that point to the end on the next call. So in-place compression of OLD messages is the worst thing for cache hit rate unless done rarely and in aligned blocks.

Config: `ModelConfig` (vibe/core/config/_settings.py:545-555) has `auto_compact_threshold`. `VibeConfig` (_settings.py) has top-level `auto_compact_threshold` (line 711), `raise_on_compaction_failure` (696), `compaction_prompt_id` (701), `safety_judge: SafetyJudgeConfig` (762) as the model for a feature-sub-config. `vibe_schema.py:65` mirrors the threshold with merge annotations. No `context_shaping` / `snip` / `microcompact` keys exist.

## Target design
Introduce a cheapest-first, in-place context-shaper pipeline that runs inside the existing `MiddlewarePipeline.run_before_turn`, BEFORE the AutoCompact stage, gated by config. Two new shaper middlewares, both pure-local (no LLM call), both mutate `context.messages` in place and return CONTINUE so the turn proceeds with a smaller history. AutoCompact stays as the last-resort fallback after them.

Ordering (cheapest -> most expensive), matching Claude Code's budget->snip->microcompact->...->auto-compact: (a) the existing per-message tool-result budget stays at ingest (already cheapest, untouched); (b) NEW SnipMiddleware — drops/elides whole OLD messages once estimated context crosses a low watermark, no compute beyond string length; (c) NEW MicrocompactMiddleware — fine-grained, cache-aware compression of the OLDEST tool results/turns in aligned blocks when the snip alone is insufficient and context is near (but below) the auto-compact threshold; (d) existing AutoCompactMiddleware — full LLM summary, last resort.

Shared base: a `ContextShaperMiddleware` Protocol-compatible base providing (1) a `_estimated_tokens(context)` helper that prefers `stats.context_tokens` when fresh but falls back to summing `approx_token_count` over message contents when `context_tokens == 0` or staleness is suspected — so shapers act on the turn the context actually grew, not one turn late; (2) `_threshold_tokens(context)` reading `active_model.auto_compact_threshold`; (3) a `_protected_prefix_len(messages)` helper returning the count of leading messages that must never be edited (system prompt at index 0, plus any compaction_context injected message, plus a cache-stability watermark — see cache notes below); (4) a `_protected_suffix_len` keeping the most-recent N turns verbatim.

SNIP semantics: when `estimated >= snip_high_watermark` (a fraction of the auto-compact threshold, default 0.6), elide content of messages in the band between the protected prefix and protected suffix, oldest-first, until `estimated <= snip_target` (default 0.5 of threshold) or the band is exhausted. "Elide" replaces a message's `content` with a short placeholder (`<vibe_snipped>… N tokens of older <role> content elided …</vibe_snipped>`) while PRESERVING message structure: keep role, keep `tool_call_id`/`name` pairing, keep `tool_calls` skeleton (so assistant->tool linkage stays valid for the backend), drop `images`. Snip targets the largest old tool results and assistant text first. Snip is idempotent: an already-snipped message is skipped (placeholder is cheap; re-snipping is a no-op).

MICROCOMPACT semantics: a middle tier that does NOT call an LLM but compresses more aggressively than snip while keeping more signal than a placeholder — it runs `truncate_middle_to_tokens(content, per_message_cap)` on the oldest tool results (default cap 2000 tokens each), collapsing only the OLDEST aligned block (a contiguous assistant+its tool messages) at a time, so the change is bounded and the rest of the prefix stays byte-identical for the provider cache. Triggered when `estimated >= microcompact_high_watermark` (default 0.8 of threshold) AND snip already ran but context is still above target. It walks oldest aligned blocks, compressing one block per turn (rate-limited) until below `microcompact_target` (default 0.7). This bounded, one-block-per-turn behavior is the cache-preservation strategy: a single contiguous edit invalidates the cache once, then the new (smaller) prefix re-caches; spreading edits across turns avoids repeated full busts.

Both shapers mutate via a new `MessageList.replace_at(index, msg)` in-place primitive (silent, no observer fire, fires nothing destructive) OR via building a new list and `reset()`. Prefer a minimal `replace_at` to avoid the reset-hooks side effects that `reset()` triggers (session/middleware reset hooks). Edits are persisted by the existing per-turn `_save_messages()` after the turn, so the on-disk log reflects the shaped history.

AutoCompact remains the safety net: if both shapers cannot get below the auto-compact threshold (e.g. the protected suffix alone exceeds it), AutoCompactMiddleware fires COMPACT as today.

## Config

- `context_shaping.snip.enabled` (bool, default `true`) — Toggle the cheap whole-message elision stage.
- `context_shaping.snip.high_watermark` (float, default `0.6`) — Fraction of auto_compact_threshold at/above which snip starts eliding old messages.
- `context_shaping.snip.target` (float, default `0.5`) — Fraction of threshold snip tries to get back down to (stop eliding once reached).
- `context_shaping.snip.keep_recent_turns` (int, default `8`) — Protected suffix: most-recent N messages never snipped (preserve working set verbatim).
- `context_shaping.snip.min_message_tokens` (int, default `300`) — Skip eliding messages smaller than this (not worth the cache churn / placeholder overhead).
- `context_shaping.microcompact.enabled` (bool, default `false`) — Toggle the mid-tier per-block compression stage (off by default; opt-in like safety_judge).
- `context_shaping.microcompact.high_watermark` (float, default `0.8`) — Fraction of threshold at/above which microcompact compresses oldest aligned blocks.
- `context_shaping.microcompact.target` (float, default `0.7`) — Fraction of threshold microcompact tries to reach.
- `context_shaping.microcompact.per_message_cap_tokens` (int, default `2000`) — Max tokens kept per compressed tool result (passed to truncate_middle_to_tokens).
- `context_shaping.microcompact.max_blocks_per_turn` (int, default `1`) — Cache-preservation rate limit: how many contiguous oldest blocks to compress per turn (1 = at most one cache bust per turn).
- `context_shaping.cache_prefix_guard_tokens` (int, default `4000`) — Never edit messages within the first N estimated tokens after the system prompt, to keep the provider's auto-cached prefix stable across edits.

## Algorithm
 1. SHARED base.before_turn(context): compute threshold = active_model.auto_compact_threshold; if threshold <= 0 return CONTINUE (shaping disabled when no threshold).
 2. SHARED _estimated_tokens(context): if stats.context_tokens > 0 use it; else (== 0, i.e. fresh/post-compact) compute sum(approx_token_count(m.content or '') for m in messages) + a small per-message overhead. Use max(stats.context_tokens, local_estimate) so a stale-low value never under-triggers.
 3. SHARED _protected_prefix_len(messages): start at 1 (system). If messages[1] is injected and looks like a compaction_context (role user, injected) include it. Then walk forward accumulating approx_token_count until cache_prefix_guard_tokens is reached; protect all messages within that band. Return the count.
 4. SHARED _protected_suffix_len(): keep_recent_turns (snip) — protect the last N messages, but expand the boundary outward so it never splits an assistant->tool group (don't strand a tool message whose assistant call was snipped).
 5. SNIP before_turn: est = _estimated_tokens(); if est < high_watermark*threshold return CONTINUE. Build editable band = messages[prefix_len : len-suffix_len]. Sort candidate indices by approx_token_count descending. For each candidate with tokens >= min_message_tokens and not already snipped: replace its content with the <vibe_snipped> placeholder (preserve role, name, tool_call_id, tool_calls skeleton; drop images) via messages.replace_at(i, new_msg); subtract saved tokens from a running est; stop once est <= target*threshold. Return CONTINUE.
 6. SNIP placeholder integrity: for an assistant message bearing tool_calls, keep the tool_calls list (ids/names) but blank arguments to '{}'; for the paired tool message keep tool_call_id and replace content. This keeps the backend's tool-call/tool-result linkage valid so the request doesn't 400.
 7. MICROCOMPACT before_turn: est = _estimated_tokens(); if est < high_watermark*threshold return CONTINUE. Identify aligned blocks (a contiguous run of [assistant-with-tool_calls, its tool results]) strictly within [prefix_len, len-suffix_len], oldest first. For up to max_blocks_per_turn oldest uncompressed blocks: for each tool message in the block, content = truncate_middle_to_tokens(content, per_message_cap_tokens); for the assistant text, also cap it; mark block compressed (e.g. tag content with a <vibe_microcompacted> sentinel so it is not re-compressed). Recompute est after each block; stop when est <= target*threshold or budget of blocks exhausted. Return CONTINUE.
 8. MICROCOMPACT cache discipline: always compress the OLDEST uncompressed block first and only one (default) per turn. This makes exactly one contiguous edit at a stable offset, so the provider re-caches the new shorter prefix next call instead of repeatedly busting. reset(COMPACT) clears the per-turn block counter.
 9. PIPELINE: run_before_turn iterates Snip then Microcompact (both return CONTINUE, no short-circuit), then AutoCompact. If after both shapers est still >= threshold, AutoCompact returns COMPACT and the existing _run_compaction path fires as today (true last resort).
 10. PERSISTENCE: shapers mutate self.messages in place; the existing post-turn `await self._save_messages()` (agent_loop.py:1167) writes the shaped history to disk. No extra save needed; no _reset_session (that's compaction-only).

## Edge cases
- context_tokens stale or 0 right after compaction or on the first turn of a prompt: must fall back to local approx estimate, else snip fires one turn late or never. Use max(stats.context_tokens, local_estimate).
- auto_compact_threshold == 0 (compaction disabled for the model): shapers must no-op (no meaningful denominator for watermarks).
- Protected suffix alone exceeds the snip target: snip can't reach target; it elides everything in the editable band and stops gracefully (no infinite loop). AutoCompact then handles the remainder.
- Assistant message with tool_calls whose paired tool message is in the protected suffix (or vice versa): never snip one without the other — boundary expansion must keep assistant+tool groups intact, otherwise the backend rejects an orphaned tool message or a tool_call with no response.
- Already-snipped / already-microcompacted message re-entering the band on a later turn: detect via sentinel tags and skip, so repeated turns don't re-process and don't churn the cache.
- Images on snipped messages: drop them (set images=None) since the content is gone; also avoids re-sending image bytes for elided turns.
- MessageList observer/reset-hook side effects: use replace_at (no notify, no reset hooks) — using reset() here would fire session/middleware reset hooks and break invariants.
- Cache-prefix guard vs aggressive watermarks: if cache_prefix_guard_tokens is large and threshold low, the editable band can be empty — shapers no-op and AutoCompact takes over; acceptable.
- Compaction_context injected message at index 1 (post-compaction state): must be in the protected prefix so a snip pass doesn't gut the just-built summary.
- Microcompact when no aligned tool blocks exist (pure chat, no tool calls): nothing to compress; no-op and let snip / autocompact handle text-heavy history.
- Plan-mode / chat-mode: shapers are content-agnostic and safe, but verify the ReadOnly reminders (injected each transition) sit in the protected suffix so they aren't elided before the model reads them.

## Test plan
- Unit: SnipMiddleware elides oldest large tool result and reduces estimated tokens below target while leaving system[0], any compaction_context, and the last keep_recent_turns messages byte-identical.
- Unit: snip preserves assistant->tool linkage — after snipping an assistant-with-tool_calls + its tool message, both retain matching tool_call_id/name and the structure validates through LLMMessage round-trip.
- Unit: snip is idempotent — running before_turn twice with the same over-threshold history produces no further edits on the second pass (sentinel detection).
- Unit: _estimated_tokens falls back to approx_token_count sum when stats.context_tokens == 0, and uses stats.context_tokens when it's larger.
- Unit: MicrocompactMiddleware compresses at most max_blocks_per_turn oldest aligned blocks per call, each tool result truncated to per_message_cap_tokens via truncate_middle_to_tokens, leaving newer blocks untouched (cache-stability assertion: prefix before the edited block is unchanged).
- Unit: protected-prefix guard — with cache_prefix_guard_tokens set, messages within the first N estimated tokens are never edited even if large.
- Unit: threshold==0 -> both shapers no-op; both return CONTINUE with messages unchanged.
- Integration (MiddlewarePipeline.run_before_turn): order Snip->Microcompact->AutoCompact; assert that when shapers bring est below threshold, AutoCompact does NOT fire COMPACT; when they cannot (suffix > threshold), AutoCompact DOES fire COMPACT.
- Integration (AgentLoop): build a synthetic over-threshold history, run one turn through act(), assert the LLM was called with a shaped (smaller) message list via _messages_for_backend and that _save_messages persisted the shaped history.
- Edge: history with an orphan-prone group at the suffix boundary — assert boundary expansion keeps the assistant+tool group whole (no orphaned tool message reaches the backend).
- Regression: with context_shaping disabled (defaults: snip on, microcompact off — or fully off), behavior matches current AutoCompact-only path; existing compaction tests still pass.
- Cache metric (optional, manual/telemetry): over a long session with generic backend auto-caching, assert cached_tokens hit-rate does not collapse turn-over-turn when microcompact runs (one-block-per-turn keeps re-caching).

## Risks
- Tool-call/tool-result pairing corruption: the highest risk. Snipping or compressing one half of an assistant->tool group, or stranding a tool message, makes the backend reject the request (400 / invalid tool sequence). Mitigation: never edit across a group boundary; expand protected suffix to whole groups; integration tests on orphan boundaries.
- Cache hit-rate regression: any edit to an old message busts the provider's auto-cached prefix from that point. If watermarks are too low or microcompact too eager, every turn re-busts the cache, raising cost despite fewer tokens. Mitigation: cache_prefix_guard_tokens, one-block-per-turn microcompact, sentinel-based idempotency so a message is edited at most once.
- Stale context_tokens causing mistimed triggers: triggering on a one-turn-late signal could over- or under-shape. Mitigated by the local approx fallback, but approx (~len/4) diverges from real tokenization, so watermarks are intentionally fractional and conservative.
- Loss of information the model still needs: snipping/compressing a tool result the model later references degrades answers. Mitigation: protect recent turns, prefer compressing (microcompact, keeps head+tail) over eliding (snip) for the near-threshold band; keep_recent_turns tunable.
- Interaction with emergency ContextTooLongError self-heal (agent_loop.py:1137-1152): if shapers shrink history but the very next backend call still overflows, the existing one-shot emergency compaction must still fire — verify shapers don't mask the overflow path or double-process.
- MessageList in-place edits vs concurrent deferred init (update_system_prompt at types.py:559-567 runs on a background thread): replace_at on non-zero indices is fine, but document that shapers only run inside act() under @requires_init, same guarantee the system-prompt update relies on.
- Config merge correctness for the new nested model across user/project config (recent commit 9ae8766 changed merge to overlay): must use the right merge annotation in vibe_schema.py so a project enabling microcompact doesn't wipe user watermarks.
